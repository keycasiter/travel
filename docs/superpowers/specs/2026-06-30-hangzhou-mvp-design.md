# 杭州单城 MVP 设计

## 目标

第一版从“8 城通用壳”收敛为“杭州完整闭环”。首页保留水墨 2.5D 中国地图和 8 个城市热点，但只有杭州进入完整内容、行程、收藏和分享链路；其它城市明确展示“待完善”，避免给用户造成全国内容已经可用的误解。

## 完成标准

| 场景 | 标准 |
|---|---|
| 首页探索 | 点击杭州出现高保真杭州卡片和完整入口；点击其它城市只提示待完善 |
| 内容厚度 | 杭州至少包含 5 个片区、12 个 POI、4 篇精选攻略、覆盖餐饮/住宿/交通/避坑/提醒的服务内容 |
| 行程规划 | 默认目的地为杭州，生成标题和每日摘要使用用户可读杭州文案 |
| 执行态 | 行程按天展示点位、交通建议、完成状态和备注 |
| 收藏 | 从杭州内容可收藏 POI、攻略和城市；收藏页能看出杭州内容归属 |
| 分享 | 只读分享展示杭州行程，接收者可复制独立副本 |
| 其它城市 | 热点保留，但卡片和入口标记为“待完善”，不进入完整详情/规划 |

## 实施边界

不新增生产后台、不接真实 AI、不做用户发布、不做真实订单。真实地图仍只在区域地图页使用微信原生 `map`，首页继续使用高质感位图。
