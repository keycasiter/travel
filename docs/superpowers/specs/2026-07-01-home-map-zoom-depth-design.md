# 首页水墨地图缩放纵深设计

## 目标

首页不使用第三方地图底图，保留高质感水墨中国地图首屏，并让缩放不只是放大同一张图。缩放过程中按照全国、杭州聚焦、杭州片区、杭州点位四个层级逐步增加本地绘制细节。

## 已确认事实

| 事实 | 证据路径 | 影响 |
|---|---|---|
| 首页底图已切到本地位图 | `apps/miniprogram/components/ink-map/index.wxml` | 本次继续沿用 `<image>` 和原生热点，不接腾讯地图 |
| 首页已有拖动、双指/按钮缩放和杭州语义点位 | `apps/miniprogram/components/ink-map/index.ts`、`apps/miniprogram/components/ink-map/home-map-layers.ts` | 本次只补视觉层级，不重写交互模型 |
| 杭州已有片区、POI、交通点数据 | `apps/miniprogram/components/ink-map/home-map-layers.ts` | 多层级展示先围绕杭州 MVP 跑通 |
| 小程序包需要控制资源大小 | `apps/miniprogram/project.config.json`、`apps/miniprogram/scripts/test-map-performance.js` | 新增贴图必须压缩，并纳入资源检查 |

## 分层方案

| 层级 | 触发区间 | 视觉内容 | 交互内容 |
|---|---|---|---|
| 全国层 | `1.00-1.04` | 完整中国水墨浮雕，低细节 | 城市热点，杭州高亮 |
| 城市聚焦层 | `1.05-1.17` | 杭州周边更清晰，外圈雾化 | 聚焦杭州，不展示片区点 |
| 片区层 | `1.18-1.34` | 叠加西湖、灵隐、湖滨等片区水墨纹理 | 展示杭州片区标记 |
| 点位层 | `1.35-1.62` | 叠加 POI、交通、避坑提示的细颗粒纹理 | 展示杭州 POI/交通点标记 |

## 资源策略

运行时使用 1 张基础图和 3 张同尺寸叠层图：

| 资源 | 用途 |
|---|---|
| `home-map-mobile.jpg` | 全国层基础底图 |
| `home-map-hangzhou-focus.jpg` | 城市聚焦层，突出杭州周边细节 |
| `home-map-hangzhou-areas.png` | 片区层透明水墨纹理 |
| `home-map-hangzhou-poi-detail.png` | 点位层透明细节纹理 |

叠层图不内嵌可读文字，城市、片区、POI 名称继续由 WXML 渲染，保证真机清晰和可点击。

## 非目标

| 不做 | 原因 |
|---|---|
| 首页街道级真实路网 | 首页是高质感探索视觉，街道地图仍从杭州卡片进入原生地图页 |
| 第三方地图 SDK 底图 | 会破坏水墨首屏风格并增加加载和卡顿风险 |
| 全国全量片区/POI | 第一版只让杭州跑通完整流程 |
| 整屏截图式 UI | 搜索框、分类按钮、热点、缩放和卡片保持原生 WXML/WXSS |

## 验收标准

| 类别 | 标准 |
|---|---|
| 视觉层级 | 缩放到杭州后可见更清晰的杭州纹理；继续放大出现片区和点位细节 |
| 性能 | 新增资源总量受脚本约束，首页不加载原生地图底图 |
| 交互 | 现有搜索、分类、点击杭州、点击片区/POI、进入街道地图仍可用 |
| 结构 | 分层资源、层级阈值和检查脚本可维护，后续可替换为更精细手绘母版 |
