# 首页中国地图高还原落地设计

## 1. 目标

把探索首页从可运行原型升级为可上线评审的首页地图体验。第一阶段聚焦首页主视觉和热点交互，不扩展新业务功能。

| 目标 | 完成状态定义 |
|---|---|
| 地图视觉高质感 | 首页使用移动端专用的高清位图母版，呈现水墨山河、玉石/宣纸质感和低饱和 3D 浮雕 |
| 地图形状可控 | 中国轮廓、台湾、海南、南海诸岛等关键区域以官方标准地图为基准，不再依赖 AI 自由生成的边界形状 |
| 热点可校准 | 8 个种子城市热点有独立坐标配置和开发态标定能力 |
| 移动端稳定 | 小屏、常规屏、高屏下全国视图、热点、搜索和底部卡片布局稳定 |
| 长期可维护 | 视觉资源、热点数据、组件交互和验收脚本拆开维护 |

## 2. 现状和约束

| 已确认事实 | 证据路径 | 影响 |
|---|---|---|
| 首页已经切换为位图主视觉，不再使用原生地图作为首页底图 | `apps/miniprogram/components/ink-map/index.wxml` | 下一步不是重新接地图 SDK，而是提升位图母版和热点精度 |
| 城市热点已在组件内实现，但坐标为人工粗标 | `apps/miniprogram/components/ink-map/index.ts` | 需要把热点数据抽离并建立标定流程 |
| 真实街道地图仍在城市/区域详情页 | `apps/miniprogram/pages/region-map/index.wxml` | 首页不承担街道级缩放、POI 和交通细节 |
| 当前位图来自 AI 生成 | `apps/miniprogram/assets/maps/china-relief-home.jpg` | 视觉可用，但公开上线前需要用标准地图锁定形状基准 |
| 小程序项目已有验证脚本 | `apps/miniprogram/scripts/test-map-performance.js` | 可把首页地图边界和资源质量写入自动检查 |

当前约束：

| 约束 | 处理 |
|---|---|
| 不依赖腾讯地图或第三方地图作为首页底图 | 首页只使用自有位图和热点配置 |
| 首页不做街道级地图 | 街道级能力保留在城市/区域详情页 |
| 第一版只覆盖 8 个种子城市 | 热点和搜索只保证北京、上海、杭州、成都、西安、广州、深圳、厦门 |
| 地图公开展示有合规要求 | 以自然资源部标准地图服务为底图基准，保留审图号和来源记录 |

## 3. 方案选择

### 3.1 推荐方案：标准地图基准 + 高质感位图渲染 + 热点校准工具

| 维度 | 设计 |
|---|---|
| 地图形状 | 使用官方标准地图作为轮廓和重要岛屿基准 |
| 视觉生产 | 在锁定轮廓上做水墨、宣纸、玉石浮雕、山脉水系质感 |
| 小程序落地 | 使用位图作为首页底图，叠加自有热点和卡片 |
| 热点精度 | 开发态点击标定坐标，生产态读取固定配置 |
| 合规边界 | 不把 AI 自由生成的中国边界直接作为最终上线底图 |

推荐原因：视觉质感、性能稳定性、地图形状可控性和后续维护成本最均衡。

### 3.2 不采用方案

| 方案 | 不采用原因 |
|---|---|
| 继续直接使用 AI 生成图 | 边界和岛屿形状不可控，热点坐标也会随着换图漂移 |
| 首页接腾讯地图并改样式 | 底图风格受 SDK 限制，性能和首屏质感不稳定 |
| 自建全国矢量地图渲染 | v1 成本过高，会转化为海量地图数据、边界精度和渲染工程问题 |

## 4. 视觉母版规范

### 4.1 资源分层

| 资源 | 用途 | 建议路径 |
|---|---|---|
| 标准地图参考源 | 形状、边界、岛屿基准 | `docs/assets/maps/standard-china-map.*` |
| 高分辨率母版 | 归档和二次裁切 | `docs/assets/maps/home-map-master.jpg` |
| 小程序首屏资源 | 实际运行加载 | `apps/miniprogram/assets/maps/home-map-mobile.jpg` |
| 资源说明 | 来源、审图号、处理方式、版本 | `docs/assets/maps/home-map-metadata.md` |

### 4.2 画面要求

| 元素 | 要求 |
|---|---|
| 中国轮廓 | 以标准地图为基准，不任意改变边界形状 |
| 台湾、海南、南海区域 | 在手机首屏可识别，不被底部卡片完全遮挡 |
| 质感 | 低饱和、水墨山河、玉石浮雕、宣纸底纹 |
| 文字 | 母版不内嵌城市名、热点、按钮或宣传文案 |
| 背景 | 不使用明显椭圆阴影、渐变光球、装饰性浮层 |
| 安全区 | 顶部搜索、分类按钮、右侧缩放、底部卡片不遮挡核心地图判断 |

### 4.3 尺寸

| 资源 | 尺寸 | 说明 |
|---|---|---|
| 母版 | 4096×5464 或更高 | 用于归档和后续裁切 |
| 小程序资源 | 1800×2401 | 当前移动端首屏运行资源规格，兼顾清晰度和 2MB 主包限制 |
| 预览图 | 1080×1440 | 用于评审、README 或视觉验收记录 |

小程序资源控制在 2MB 以内。若视觉质量不足，优先调 JPEG 质量和局部锐化，不直接扩大运行资源尺寸。

## 5. 热点和交互设计

### 5.1 热点数据模型

城市热点从组件源码中抽离到独立配置：

```ts
export interface CityHotspot {
  id: string;
  name: string;
  x: number;
  y: number;
  recommendedDays: string;
  mood: string;
  summary: string;
  tags: string[];
  notes: Record<'landmark' | 'scenic' | 'food' | 'transport' | 'inspiration', string>;
}
```

坐标使用百分比坐标，`x=0/y=0` 表示底图左上角，`x=100/y=100` 表示底图右下角。热点配置与位图版本绑定，换图必须重新标定。

### 5.2 标定模式

开发态提供热点标定开关：

| 能力 | 行为 |
|---|---|
| 显示十字线 | 点击或拖动时显示当前位置参考线 |
| 输出坐标 | 点击地图后输出 `x`、`y` 百分比到控制台 |
| 复制配置片段 | 输出可直接粘贴到热点配置的对象片段 |
| 关闭生产入口 | 标定入口不出现在生产 UI |

标定模式只解决城市点位校准，不作为用户功能。

### 5.3 首页交互

| 操作 | 行为 |
|---|---|
| 轻微拖动 | 在限定范围内移动底图，不产生无限地图错觉 |
| 缩放 | 控制在 `1.0x-1.35x`，用于增强探索感 |
| 点击城市热点 | 聚焦热点并打开城市探索卡片 |
| 搜索城市 | 只搜索 8 个种子城市和目的地别名 |
| 点击分类 | 切换当前城市卡片内的地标、景观、美食、交通、灵感提示 |
| 看城市内容 | 打开已有城市内容底部卡片 |
| 街道级地图 | 从城市内容卡片进入区域真实地图页 |

首页不展示腾讯 POI 搜索结果、路线预览、街道级缩放或交通站点高亮。

## 6. 合规和来源记录

### 6.1 标准地图来源

地图形状以自然资源部标准地图服务系统为基准。资源说明文件记录：

| 字段 | 内容 |
|---|---|
| 来源 URL | `http://bzdt.ch.mnr.gov.cn/` |
| 地图名称 | 下载的标准地图名称 |
| 审图号 | 标准地图标注的审图号 |
| 下载日期 | 资源进入仓库的日期 |
| 处理方式 | 裁切、调色、材质叠加、浮雕渲染等 |
| 是否改变底图 | 明确说明是否改变边界、岛屿、附图或文字信息 |

直接使用标准地图需要标注审图号。对地图进行裁切、修改、重新渲染等编辑后公开使用，需在资源说明中标记风险，并在上线前确认是否需要送自然资源主管部门审核。

### 6.2 公开上线边界

| 场景 | 处理 |
|---|---|
| 只作为内部原型 | 可使用当前 AI 风格图继续验证交互 |
| 对外展示或上线 | 需要使用标准地图基准重新制图，并保留审图号和来源说明 |
| 视觉处理改变了底图边界或重要地理元素 | 需要重新评估地图审核要求 |
| 母版移除标准地图文字标注 | 需确认审图号展示和来源说明放置方式 |

## 7. 文件和代码边界

| 模块 | 改动方向 |
|---|---|
| `components/ink-map` | 保持首页地图组件，只负责位图、热点、搜索和城市卡片 |
| `components/ink-map/city-hotspots.ts` | 新增，存放热点配置和城市探索短文案 |
| `apps/miniprogram/assets/maps` | 存放小程序运行时地图资源 |
| `docs/assets/maps` | 存放标准地图参考源、高分辨率母版和资源说明 |
| `scripts/test-map-performance.js` | 扩展检查：禁止首页重新引入 `<map>`、腾讯搜索、超大资源、缺失热点 |
| `pages/region-map` | 不改动，继续承担真实地图、POI 和交通细节 |

## 8. 验收标准

| 类别 | 标准 |
|---|---|
| 视觉 | 首页第一屏显示高清 2.5D 中国地图，无 CSS 方案图感 |
| 准确性 | 8 个城市热点肉眼落点正确，台湾、海南、南海区域可识别 |
| 性能 | 真机滑动和缩放不卡死，首页不再加载腾讯地图底图 |
| 适配 | 小屏、常规屏、高屏均不出现热点严重偏移或文字重叠 |
| 结构 | 热点数据独立文件维护，换图时有标定流程 |
| 验证 | 小程序 typecheck、组件事件检查、首页地图约束测试通过 |

## 9. 实施顺序

1. 建立标准地图资源和元数据记录。
2. 生成或制作高质感移动端地图母版 v2。
3. 抽离城市热点配置。
4. 增加开发态热点标定模式。
5. 用标定模式校准 8 个城市。
6. 扩展首页地图约束测试。
7. 做真机视觉验收并记录剩余偏差。

实施完成后，首页仍保持“探索 / 城市卡片 / 行程入口”的产品边界，不新增街道级首页地图能力。
