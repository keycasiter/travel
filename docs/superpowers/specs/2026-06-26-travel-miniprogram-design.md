# 旅游攻略微信小程序 MVP 设计

## 1. 目标

构建一个面向中国境内自由行用户的微信小程序。第一版服务两个核心场景：

| 场景 | 用户目标 | 产品能力 |
|---|---|---|
| 出发前做攻略 | 快速理解目的地、筛选兴趣点、生成可执行行程 | 3D 探索地图、官方精选内容、规则行程生成、收藏和分享 |
| 到达后按天执行 | 按日程推进旅行，查看提醒、预算、交通和避坑 | 行程执行态、旅行管家底座、完成状态和备注 |

第一版以本地可运行 MVP 为完成标准：小程序、Go API 和 MySQL 真实连通，内容使用官方种子数据，不要求生产部署、监控、灰度和运营后台。

## 2. 决策证据表

| 已确认结论 | 依据 | 影响 |
|---|---|---|
| 运行形态为微信小程序 | 项目目标明确为微信小程序 | 前端采用原生微信小程序 + TypeScript，不做 H5 或多端框架 |
| 首页是水墨风格 3D 中国探索地图 | 首屏需要全屏吸引力，可缩放、平移、定位 | 首页承担探索和内容发现，不作为普通列表页 |
| 行程规划独立为 tab | 行程规划需要完整交互，不应塞进首页 | 底部 tab 包含探索、行程、收藏、我的 |
| 内容范围为中国境内 | 目标覆盖中国境内自由行 | 地图视觉覆盖全国，种子数据先覆盖 8 个城市 |
| 第一版不接真实 AI | 行程生成需先保证可控和可跑通 | 后端用规则 + 种子数据生成行程，预留后续替换能力 |
| 前后端在同一个 Git 仓库，API 交互 | 架构边界明确为前后端分离 | 采用 monorepo，前端和 Go API 分目录 |
| 后端为 Go + Gin + GORM + MySQL | 后端技术栈已确认 | 使用 MySQL 8.0、GORM、golang-migrate 和 REST JSON API |
| 第一版官方种子内容，不开放用户发布 | 内容治理边界已确认 | 第四层命名为精选攻略，不做 UGC 发布、评论和审核 |

## 3. 产品信息架构

底部 tab 固定为四个入口：

| Tab | 职责 |
|---|---|
| 探索 | 3D 中国地图、行政区划钻取、内容发现、地图热点详情 |
| 行程 | 行程规划、按天编辑、执行态和旅行管家底座 |
| 收藏 | 收藏景点、攻略、路线和片区 |
| 我的 | 授权状态、已保存行程、设置 |

探索首页上方或地图浮层提供“规划行程”快捷按钮，点击后切换到行程 tab。首页不承载完整规划流程。

内容层级按空间容器组织：

| 层级 | 内容 | 第一版边界 |
|---|---|---|
| 第一层 | 行政区划 | 默认城市，模型支持到末级行政区划 |
| 第二层 | 旅行便利服务 | 餐饮、住宿、交通；交通拆分为大交通、小交通、公共交通、自驾 |
| 第三层 | 兴趣点 | 景点、地标、打卡点 |
| 第四层 | 精选攻略 | 官方种子内容，不开放用户发布 |

第一版地图视觉覆盖全国。真实内容覆盖 8 个种子城市：北京、上海、杭州、成都、西安、广州、深圳、厦门。数据模型支持末级行政区划，首版真实内容填充到种子城市的区县或热门片区级；只有有内容的区域展示服务、景点和攻略。

## 4. 核心用户流程

### 4.1 首次进入和定位

首次进入先展示全屏水墨 3D 中国地图，不强制微信资料授权。页面随后展示轻量定位授权引导：

| 用户选择 | 体验 |
|---|---|
| 同意定位 | 地图相机飞到用户所在地附近，展示附近城市或内容提示 |
| 拒绝定位 | 保持全国视角，展示城市搜索和已支持城市入口 |

用户身份使用 `wx.login` 建立后端会话。保存行程、收藏和分享时绑定用户数据；头像昵称和手机号授权不作为第一版要求。

### 4.2 探索内容

用户在地图中缩放、平移和点击城市、片区或热点。点击后先打开底部半屏卡片，卡片展示：

| 内容 | 说明 |
|---|---|
| 区域概览 | 行政区或片区的摘要、标签和推荐理由 |
| 分类入口 | 餐饮、住宿、交通、景点、精选攻略 |
| 推荐点位 | 可收藏或加入行程的 POI |
| 关键提示 | 便利信息、避坑和旅行注意事项 |
| 操作 | 收藏、加入行程、进入详情页 |

需要深读时进入详情页。直接跳转详情页不是默认交互，以免打断地图探索。

### 4.3 行程规划

行程规划入口来自行程 tab 和探索首页的快捷按钮。第一版使用三步轻向导：

| 步骤 | 输入 |
|---|---|
| 目的地 | 支持的种子城市或片区 |
| 天数/日期 | 出行天数，日期可选 |
| 偏好 | 历史文化、亲子、城市漫步、美食、轻松、打卡等 |

后端基于规则和种子数据生成初始日程，不调用真实 AI。生成后用户可以：

| 能力 | 第一版行为 |
|---|---|
| 调整顺序 | 支持同一天内点位顺序调整 |
| 删除点位 | 从日程中移除点位 |
| 替换点位 | 从同区域或同偏好候选中替换 |
| 添加备注 | 为行程项添加用户备注 |
| 查看交通建议 | 展示点位间交通提示 |

复杂拖拽跨天、自动重排、实时协作和专业路线优化不进入第一版范围。

### 4.4 旅行管家底座

行程生成后进入执行态。第一版管家底座包含：

| 能力 | 说明 |
|---|---|
| 天气摘要 | 后端提供天气接口契约，首版返回 mock 或静态摘要 |
| 预算记录 | 记录预算小计和用户输入支出 |
| 今日提醒 | 展示当天注意事项和关键时间提示 |
| 通知 | 按微信小程序能力设计订阅或页面内提醒 |
| 交通建议 | 展示大交通、小交通、公共交通、自驾建议 |
| 完成状态 | 点位打勾、按天查看进度 |
| 避坑提示 | 展示预约、拥挤、低性价比和绕路提醒 |

票务、酒店订单管理、实时导航、位置触发提醒和交易闭环不进入第一版范围。

### 4.5 分享和复制

分享使用后端 `shareCode` 快照。微信好友打开后进入小程序内只读行程页，展示封面、目的地、天数、每日安排、预算摘要和避坑提示。

接收者可以收藏副本。收藏时复制一份独立行程到接收者账号，不与原作者后续编辑同步，也不能编辑原作者行程。

## 5. 技术架构

### 5.1 Monorepo 结构

建议目录：

```text
apps/
  miniprogram/
  api/
packages/
  shared/
data/
  seeds/
docs/
  superpowers/
    specs/
```

| 目录 | 职责 |
|---|---|
| `apps/miniprogram` | 原生微信小程序 + TypeScript |
| `apps/api` | Go API 服务 |
| `packages/shared` | OpenAPI、共享类型、种子数据 schema 或生成物 |
| `data/seeds` | 官方种子内容 |
| `docs` | 设计、接口和启动说明 |

### 5.2 前端

前端采用原生微信小程序 + TypeScript，不引入 Taro 或 uni-app。

水墨 3D 中国地图第一版采用自定义 Canvas/WebGL 轻量渲染，实现风格化中国地图、城市热点、缩放、平移和定位飞行动效。地图视觉只追求省/重点城市级热点和品牌化探索体验，不追求真实行政边界精确 3D 渲染。

真实地图、定位和路线相关能力由微信定位、微信 `map` 组件和腾讯位置服务承接。探索首页的 3D 视觉层和真实地图能力保持分层。

### 5.3 后端

后端采用：

| 技术 | 用途 |
|---|---|
| Go | API 服务语言 |
| Gin | HTTP 路由和中间件 |
| GORM | MySQL 数据访问 |
| MySQL 8.0 | 业务数据存储 |
| golang-migrate | 数据库迁移 |
| OpenAPI | REST JSON 接口文档 |

后端职责：

| 模块 | 能力 |
|---|---|
| 用户 | `wx.login` 换取后端会话，保存用户身份 |
| 内容 | 行政区、服务、POI、精选攻略查询 |
| 行程 | 规则生成、CRUD、按天编辑、执行状态 |
| 收藏 | 收藏和取消收藏 |
| 分享 | 生成 `shareCode` 快照，读取只读分享，复制副本 |

### 5.4 本地开发环境

本地环境使用本机 MySQL + 本机 Go API。默认数据库版本为 MySQL 8.0，库名建议为 `travel_app`，字符集为 `utf8mb4`。

仓库提供：

| 文件或命令 | 作用 |
|---|---|
| `.env.example` | 示例连接配置 |
| 迁移命令 | 初始化和更新 MySQL schema |
| seed 导入命令 | 将官方内容导入 MySQL |
| API 启动命令 | 本机启动 Go 服务 |
| 小程序启动说明 | 使用微信开发者工具打开 `apps/miniprogram` |

真实密码不写入仓库，连接信息走 `.env`。

## 6. 核心数据模型

| 模型 | 关键字段 | 用途 |
|---|---|---|
| `Region` | `id`、`name`、`level`、`parentId`、`center`、`bounds`、`enabled` | 城市、区县、片区和末级行政区划 |
| `TravelService` | `regionId`、`type`、`title`、`summary`、`tips` | 餐饮、住宿、交通等便利信息 |
| `Poi` | `regionId`、`type`、`name`、`location`、`tags`、`duration`、`costLevel` | 景点、地标、打卡点 |
| `Guide` | `regionId`、`title`、`content`、`tags`、`cover`、`official` | 精选攻略 |
| `Itinerary` | `userId`、`destination`、`days`、`status`、`budget`、`shareCode` | 用户行程 |
| `ItineraryDay` | `itineraryId`、`dayIndex`、`date`、`summary` | 每日安排 |
| `ItineraryItem` | `dayId`、`poiId`、`startHint`、`duration`、`transportHint`、`note`、`done` | 每个点位 |
| `Favorite` | `userId`、`targetType`、`targetId` | 收藏 |
| `ShareSnapshot` | `shareCode`、`itinerarySnapshot`、`expiresAt` | 只读分享快照 |

种子内容以文件形式维护，通过后端命令导入 MySQL。运行时查询以 MySQL 为准，前端不内置正式业务内容。

## 7. API 边界

API 使用 REST JSON + 统一错误码 + OpenAPI 文档。

建议核心接口：

| 方法 | 路径 | 用途 |
|---|---|---|
| `POST` | `/api/v1/auth/wechat-login` | 微信登录换取后端会话 |
| `GET` | `/api/v1/regions` | 查询行政区划 |
| `GET` | `/api/v1/regions/:id/overview` | 查询区域概览 |
| `GET` | `/api/v1/regions/:id/services` | 查询区域旅行便利服务 |
| `GET` | `/api/v1/regions/:id/pois` | 查询区域 POI |
| `GET` | `/api/v1/regions/:id/guides` | 查询区域精选攻略 |
| `POST` | `/api/v1/itineraries/generate` | 规则生成行程 |
| `GET` | `/api/v1/itineraries` | 查询用户行程列表 |
| `GET` | `/api/v1/itineraries/:id` | 查询行程详情 |
| `PATCH` | `/api/v1/itineraries/:id` | 更新行程基础信息 |
| `PATCH` | `/api/v1/itinerary-items/:id` | 更新行程项状态、备注或顺序 |
| `POST` | `/api/v1/favorites` | 创建收藏 |
| `DELETE` | `/api/v1/favorites/:id` | 取消收藏 |
| `POST` | `/api/v1/itineraries/:id/share` | 创建分享快照 |
| `GET` | `/api/v1/shares/:shareCode` | 读取只读分享 |
| `POST` | `/api/v1/shares/:shareCode/copy` | 复制分享行程为独立副本 |
| `GET` | `/api/v1/weather/summary` | 获取天气摘要 |

## 8. 实施路线

采用垂直闭环优先。

| 阶段 | 目标 | 完成标准 |
|---|---|---|
| 1 | 打通 1-2 个城市的完整链路 | MySQL schema、seed、内容 API、探索页、行程生成和详情可跑 |
| 2 | 补齐行程编辑、执行、收藏和分享复制 | 用户可以完成规划到执行的闭环 |
| 3 | 扩充到 8 个种子城市 | 每城具备片区、服务、POI 和精选攻略 |
| 4 | 完善文档和验收 | README、OpenAPI、测试和手动验收步骤完整 |

先打通完整链路，再横向扩充城市内容，避免页面或数据任一侧长期停留在假闭环。

## 9. 验收标准

| 类别 | 标准 |
|---|---|
| 小程序 | 微信开发者工具打开 `apps/miniprogram` 后可运行 |
| 探索首页 | 展示全屏水墨 3D 中国地图，支持缩放、平移、城市热点、定位授权和拒绝分支 |
| 内容 | 可浏览 8 个种子城市的行政区、服务、景点、地标和精选攻略 |
| 行程 | 可生成、编辑、执行、收藏、分享和复制只读分享副本 |
| 后端 | Go API 连接本机 MySQL，迁移和 seed 导入可执行 |
| 接口 | REST JSON 可用，关键接口有 OpenAPI 描述 |
| 测试 | 后端关键路径测试通过，前端完成构建和核心手动验收 |
| 文档 | README 写清本机 MySQL、迁移、seed、API、小程序启动和验收步骤 |

## 10. 第一版范围外

| 范围外事项 | 原因和影响 |
|---|---|
| 真实 AI 行程生成 | 规则生成更可控，后续可在后端替换生成实现 |
| 用户发布、评论和社区关系 | 第一版内容来自官方种子，避免审核和内容治理复杂度 |
| 管理后台 | seed 文件和导入命令满足 MVP 内容维护 |
| 支付、票务、酒店和餐饮预订 | 交易会引入资质、订单、售后和合规复杂度 |
| 实时协作编辑 | 只读分享和复制副本满足第一版传播闭环 |
| 实时导航和位置触发提醒 | 权限、稳定性和地图能力边界复杂 |
| 生产部署、监控和灰度 | 第一版完成标准是本地完整闭环可跑通 |

## 11. 待确认项

| 项目 | 影响 |
|---|---|
| 腾讯位置服务 key 和使用额度 | 影响真实地图、路线和位置服务接入 |
| 种子内容来源和图片授权 | 影响 8 个城市内容质量和合规 |
| 微信小程序 AppID | 影响登录、定位权限、订阅消息和真机调试 |
| 天气接口生产供应商 | 第一版使用 mock 或静态摘要，生产化前需替换 |
